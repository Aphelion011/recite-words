# 贡献指南 CONTRIBUTING
欢迎参与 recite-words 背单词助手开源项目开发！
本文档说明参与项目开发、提交代码、提交Issue、PR审查全流程规范。

## 1. 开发前准备
1. Fork 本仓库到个人GitHub账号
2. 克隆仓库到本地：
git clone https://github.com/Aphelion011/recite-words.git
3. 本地新建分支开发，禁止直接在main分支修改代码
分支命名规范：
- 功能开发：feature/词汇朗读功能
- Bug修复：fix/正确率计算bug
- 文档优化：docs/完善README说明
- UI美化：style/调整按钮配色

## 2. Issue 提交流程
1. 新增功能、优化建议、bug反馈都先新建Issue；
2. Issue标题简洁清晰，正文写明需求/问题、复现步骤、预期效果；
3. 自行认领Issue后，在Issue内留言告知负责人；
4. 开发完成后关联对应Issue编号提交PR。

## 3. Commit 提交规范
提交注释统一格式：`类型: 简要描述`
类型可选：
- feat：新增功能
- fix：修复bug
- docs：修改文档README
- style：页面样式、UI调整
- data：词库数据新增/修改
示例：
feat: 新增雅思词汇拓展
fix: 修复切换词库统计不清零问题
style: 优化按钮配色适配主题

## 4. Pull Request 提交规范
1. 开发完成后向main分支提交PR；
2. PR标题写明改动内容，正文关联对应Issue #编号；
3. 提交前自测全部功能，保证页面在Chrome/Edge/Safari正常显示；
4. PR必须经过Reviewer代码审查，收到修改意见需及时更新提交；
5. Review通过后由组长合并分支。

## 5. 代码审查（Review）标准
审查人员检查以下内容：
1. 代码无语法错误，不破坏原有功能；
2. 词库数据格式统一，无乱码、缺失释义；
3. UI样式与整体页面风格统一；
4. 注释清晰，新增逻辑易懂；
5. 无冗余无效代码。

## 6. 自测要求（提交前必须完成）
1. 四级/六级/考研/雅思四组词库切换测试；
2. “认识/不认识”计数、正确率统计校验；
3. 重置、背诵完成弹窗功能验证；
4. 多浏览器兼容测试。
